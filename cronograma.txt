CRONOGRAMA DE DESENVOLVIMENTO DO PROJETO

------------------------------------------------------------
FASE I – PLANEJAMENTO E ESTRUTURA INICIAL
Responsável: Membro 1 (Líder do Repositório)
Período: Dia 1

Objetivos:
- Criar o repositório vazio no GitHub: techwave-site
- Adicionar arquivos iniciais:
  - README.md com a descrição do projeto
  - equipe.txt com o nome e papel do Membro 1
- Fazer o primeiro commit e push para o repositório remoto
- Compartilhar o link do repositório com os Membros 2 e 3

Entrega esperada:
Repositório configurado e acessível para todos
------------------------------------------------------------


FASE II – CONFIGURAÇÃO E DEFINIÇÃO DAS BRANCHES
Responsáveis: Membros 2 e 3
Período: Dia 2

Objetivos:
- Clonar o repositório para os ambientes locais
- Criar suas respectivas branches:
  Membro 2 → feature-funcionalidades
  Membro 3 → feature-tecnologias
- Membro 1 cria o arquivo cronograma.txt e adiciona à main

Entrega esperada:
Branches criadas e sincronizadas
------------------------------------------------------------


FASE III – DESENVOLVIMENTO INICIAL
Responsáveis:
  Membro 2 (Front-end)
  Membro 3 (Back-end / Infraestrutura)
Período: Dias 3 e 4

Membro 2 – feature-funcionalidades
Tarefas:
- Criar o arquivo funcionalidades.txt com o plano do front-end:
  • Estrutura básica em HTML
  • Estilo com CSS moderno (Flexbox/Grid)
  • Layout das páginas principais (Home, Sobre, Contato)
  • Implementar design responsivo
- Fazer pelo menos 2 commits descritivos:
  “Adiciona estrutura inicial de páginas HTML”
  “Adiciona layout responsivo e menu de navegação”

Membro 3 – feature-tecnologias
Tarefas:
- Criar o arquivo tecnologias.txt com as tecnologias do projeto:
  • Front-end: HTML5, CSS3, JavaScript (ou React)
  • Back-end: Node.js com Express
  • Banco de dados: MongoDB
  • Versionamento: Git/GitHub
  • Implantação: Vercel ou Netlify
- Fazer pelo menos 2 commits descritivos:
  “Lista tecnologias base do projeto”
  “Inclui stack de deploy e banco de dados”

Entrega esperada:
Branches atualizadas com commits visíveis e arquivos criados
------------------------------------------------------------


FASE IV – INTEGRAÇÃO E TESTE DE CONFLITOS
Responsáveis: Todos os membros
Período: Dia 5

Passos:
1. Merge de funcionalidades (Membro 1):
   - Fazer git pull e git merge feature-funcionalidades na main
2. Criação de conflito intencional:
   - Membro 1 e Membro 3 alteram a mesma linha do arquivo equipe.txt
3. Resolução de conflito:
   - Qualquer membro resolve o conflito e faz commit de resolução

Entrega esperada:
Merge realizado, conflito resolvido, histórico limpo e rastreável
------------------------------------------------------------


FASE V – REVISÃO FINAL E ENTREGA
Responsável: Membro 1
Período: Dia 6

Passos:
- Confirmar que todos os arquivos (funcionalidades.txt, tecnologias.txt, equipe.txt e cronograma.txt) estão atualizados
- Fazer o push final para a branch principal
- Aplicar tag de versão:
    git tag v1.0.0
    git push --tags
- Garantir que o histórico de commits mostre a contribuição dos três membros

Entrega esperada:
Repositório final com versão v1.0.0 publicada
------------------------------------------------------------


RESUMO DAS FUNÇÕES DA EQUIPE

Membro 1 - Líder de Projeto / Controle de Versionamento
  • Criar repositório, revisar merges, resolver conflitos, aplicar tag final

Membro 2 - Front-end Developer
  • Criar estrutura HTML/CSS, layout e funcionalidades da interface

Membro 3 - Back-end Developer / DevOps
  • Listar tecnologias, preparar estrutura de servidor, resolver conflitos e integração
------------------------------------------------------------


OBSERVAÇÕES FINAIS
- Todos os commits devem ter mensagens claras e explicativas
- O repositório deve mostrar o histórico de merges e conflitos resolvidos
- O cronograma pode ser ajustado conforme o progresso da equipe
